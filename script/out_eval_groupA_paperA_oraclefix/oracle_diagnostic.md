# Oracle Branch Diagnostic

## 1. Inference and Reconstruction Path

- Oracle labels are generated by `oracle_lib.oracle_label_w`.
- For each frame and each macroblock, the solver enumerates discrete weights in `w_set` and picks the minimum-loss weight.
- In the fixed implementation, oracle reconstruction uses the same temporal blending form as other methods and uses oracle recursive reference.

## 2. Oracle Objective Function

- The objective is macroblock-level reconstruction error minimization, not direct frame-level PSNR/SSIM maximization.
- Concretely, each candidate weight is applied with temporal IIR blending, then compared with ground truth using per-pixel absolute error aggregated at macroblock level.
- Macroblock size in this run is `4x4`.

## 3. Metric-Path Consistency Check

- Decision-level oracle is evaluated against oracle labels generated under the same oracle branch path, so its decision metrics are identically optimal.
- Image-level metrics are computed after frame reconstruction and therefore depend on objective mismatch and sequence dynamics.

## 4. Interpretation of Decision-Level 1.0 vs Image-Level Metrics

- Decision-level score 1.0 means the branch exactly predicts the labels used as supervision target.
- This does not guarantee global optimum for PSNR/SSIM because the label objective is local macroblock MAE on a discrete set.
- Frame-level PSNR/SSIM are nonlinear global metrics over all pixels and channels, and may diverge from local MAE optimum.
- Therefore, oracle must be interpreted as an upper bound with respect to its verified local objective, not a universal upper bound for every image-level metric.
